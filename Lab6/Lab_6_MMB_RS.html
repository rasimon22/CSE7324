1. Introduction¶

In this lab, we use a convolutional neural network model to classify a given 2-D autocorrelation image into one of the 10 numeric digits. A huge number of imaging techniques such as non-line of sight imaging, coherent X-ray diffraction imaging, electron microscopy,astronomy and super-resolution optical imaging [1,2] rely on the ability to recover the image of the object from its autocorrelation. The focus in this work is on the non line of sight imaging problem, wherein, the image of an object hidden around a corner needs to be extracted at sub-millimeter resolutions.

1.1. Indirect imaging correlography[3]¶

The hidden object is indirectly illuminated by the light scatterred off of a rough surface (wall) as shown in figure below. The light after scatterring off of the object finds it way to the observable region on the wall, known as virtuial detector. Due to the random nature of the rough surfaces involved, the resulting pattern at the virtual detector will have a granular appearance, also known as a speckle pattern. This pattern is recorded by a camera imaging the wall. By exploiting the properties of this speckle pattern, the autocorrelation of the object can be estimated.

The inability to directly get at the object's image from these speckle patterns is due to the fact that only the intensity of the light is recorded by the camera and the phase information (which is a function of the total path traveled by the light) is absent. Thus, traditionally, a class of algorithms known as phase retrieval algorithms are employed to retrieve the image of the object from its autocorrelation. But, these phase retrieval algorithms require very high fidelity measurements with less noise.

Since the object of interest will be a part of an alphabet or a digit, for eg., on an ID badge, we can try to build a classification algorithm that can detect a digit/alphabet or multiple digits/alphabet using a CNN. In this lab, we only focus on classifying digits given an autocorrelation.

1.2. Training dataset¶

To train the classifier, we generated the autocorrelation from the MNIST database[1]. To generate these autocorrelations, we zero-padded the images to twice the actual size. From these padded images, we generated the autocorrelations and saved it to a mat-file. The matlab code used to generate these autocorrelations from MNIST dataset is shared in the appendix.

The MNIST images are of size 28 X 28. Our autocorrelations are of size 56 X 56.

Thus, the inputs for the classifier are the autocorrelation images and the outputs are the one-hot encoded labels from MNIST database.

MNIST dataset has 60,000 training examples and 10,000 test examples. And these dataset is pretty uniformly distributed across all the digits. Hence, we decided not to do any further splitting of data.

We have tried two CNN architectures with this dataset which include: 1) A normal CNN and 2) A ResNet with skip connections

We used accuracy as the metric to compare the performance of the networks. The dataset is uniformly distributed across all the in both training and testing dataset. And, we also expect the classifier to classify all the digits with equal accuracy as no one digit is more likely than the other in the scenario mentioned in section-1.

Although the dataset is pretty extensive, we still used data expansion, by using rotation, translation, zoom functions to genrate more images for the classification task.

def summarize_net(net, X_test, y_test, title_text=''):
    plt.figure(figsize=(15,5))
    yhat = np.argmax(net.predict(X_test), axis=1)
    acc = mt.accuracy_score(y_test,yhat)
    cm = mt.confusion_matrix(y_test,yhat)
    cm = cm/np.sum(cm,axis=1)[:,np.newaxis].astype(np.float16)
    sns.heatmap(cm, annot=True, fmt='.2f')
    plt.title(title_text+'{:.4f}'.format(acc))

2. Convolutional Neural Network¶

A CNN is implemented in this section using Keras. The CNN model follows the standard CNN used in the cnn-demo used in class. It has two convolutional layers with 32 channels (filters to learn) and 64 channels respectively followed by a fully connected layer with softmax activation to generate the 10-dimensional output determining the class to which the autocorelation belongs.

We ran this on the entire training set on a GPU for 20 epochs. We have used a batch size of 256 to train the network.

cnn = Sequential()

# let's start with an AlexNet style convolutional phase
cnn.add(Conv2D(filters=32,
                input_shape = (img_wh,img_wh,1),
                kernel_size=(3,3), 
                padding='same', 
                activation='relu', data_format="channels_last")) # more compact syntax

# no max pool before next conv layer!!
cnn.add(Conv2D(filters=64,
                kernel_size=(3,3), 
                padding='same', 
                activation='relu')) # more compact syntax

# add one layer on flattened output

cnn.add(Flatten())

cnn.add(Dropout(0.5)) # add some dropout for regularization, again!
cnn.add(Dense(NUM_CLASSES, activation='softmax'))

# Let's train the model

One interesting confusion to note from this confusion matrix is the confusion between 6, 9. As the autocorrelation is a symmetric function, the autocorrelation of 6 and 9 would be almost identical. A point to be noted here is that, although we know this issue exists, these confusions are not accepatble for us and hence went for the accuracy metric.

# now lets use the LeNet architecture with batch norm
# We will also use ReLU where approriate and drop out 
    from keras.layers.normalization import BatchNormalization
    from keras.layers import Add
    from keras.layers import average, concatenate
    from keras.models import Input, Model
    from keras.regularizers import l2 
    l2_lambda = 0.0001

    input_holder = Input(shape=(img_wh, img_wh, 1))

    # start with a conv layer
    x = Conv2D(filters=32,
                   input_shape = (img_wh,img_wh,1),
                   kernel_size=(3,3),
                   kernel_initializer='he_uniform', 
                   kernel_regularizer=l2(l2_lambda),
                   padding='same', 
                   activation='relu', 
                   data_format="channels_last")(input_holder)

    x = MaxPooling2D(pool_size=(2, 2), data_format="channels_last")(x)

    x = Conv2D(filters=32,
                   kernel_size=(3,3),
                   kernel_initializer='he_uniform', 
                   kernel_regularizer=l2(l2_lambda),
                   padding='same', 
                   activation='relu', 
                   data_format="channels_last")(x)

    x_split = MaxPooling2D(pool_size=(2, 2), data_format="channels_last")(x)

    x = Conv2D(filters=64,
                   kernel_size=(1,1),
                   kernel_initializer='he_uniform', 
                   kernel_regularizer=l2(l2_lambda),
                   padding='same', 
                   activation='relu', 
                   data_format="channels_last")(x_split)

    x = Conv2D(filters=64,
                   kernel_size=(3,3),
                   kernel_initializer='he_uniform', 
                   kernel_regularizer=l2(l2_lambda),
                   padding='same', 
                   activation='relu', 
                   data_format="channels_last")(x)

    x = Conv2D(filters=32,
                   kernel_size=(1,1),
                   kernel_initializer='he_uniform', 
                   kernel_regularizer=l2(l2_lambda),
                   padding='same', 
                   activation='relu', 
                   data_format="channels_last")(x)

    # now add back in the split layer, x_split (residual added in)
    x = Add()([x, x_split])
    x = Activation("relu")(x)

    x = MaxPooling2D(pool_size=(2, 2), data_format="channels_last")(x)

    x = Flatten()(x)
    x = Dropout(0.25)(x)
    x = Dense(256)(x)
    x = Activation("relu")(x)
    x = Dropout(0.5)(x)
    x = Dense(NUM_CLASSES)(x)
    x = Activation('softmax')(x)

    resnet = Model(inputs=input_holder,outputs=x)

    resnet.summary()

4. MLP¶

An MLP is implemented in this section using Keras. This model also follows the cnn-demo used in class. We had 6-hidden layers with neurons varying from 1024 to 4096 across layers. We had over 29 million parameters in this network.

Despite the complexity, the MLP performed terribly on the dataset with always classifying the data into class - 1.

We believe this is due to the absence of convolutional layers to extract the features present the autocorrelations. Just a normal MLP is unable to learn the rich features present in the data to classify and ends up always detecting a 1.

Summary¶

In summary, for this lab, we have explored CNNs and ResNet architectures to classify autocorrelations of digits. CNNs performed exceedingly well on the whole with obtaining 95.5% accuracy within 20 epochs and using 4 convolutional layers. ResNet could get upto 84% within the same time but could have converged had we let it run for a few more epochs. CNNs preformed very well across all classes barring 6 and 9. In hindsight, we think, we could have penalized more for these 6,9 confusions compared to others to make the CNN run better and obtained a much higher accuracy. ResNet had several more such confusions but it most likely would have converged eventually.

MLP was terrible and is as good as a random guess and transfer learning approach struggled to converge because the weights of ResNet 50 were optimised for natural images. Since we used autocorrelations, the features extracted by convolutional layers of ResNet couldn't really explain the structures present in autocorrelation images.

%%
sz = size(images);
data = zeros(sz(2),56,56);
for i = 1:sz(2)
    i
    temp_label = labels(i,1);
    temp = images(:,i);
    temp = reshape(temp,[28,28]);
    temp = padarray(temp,[14 14],'both');% pad array to so that the autocorrelation is not aliased
    
    % estimate autocorrelation using FFTs
    
    temp_propagated = (abs(fftshift(fft2(temp)))).^2; 
    backprop = abs(fftshift(fft2(temp_propagated)));
    autocorr = backprop.^2;
    
    % Normalization
    autocorr = 255*(autocorr - min(autocorr(:)))./(max(autocorr(:)) - min(autocorr(:)));

%     % Write to image
%     str = ['./test/' sprintf('image%d_',i) num2str(temp_label) '.png'];
%     imwrite(autocorr,str)
    data(i,:,:) = uint8(autocorr);
end
